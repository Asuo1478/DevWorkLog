# 研发工作流平台 (DevWorkLog) 生产环境部署文档

本文档详细介绍了 `dev-work-log-server`（后端）和 `dev-work-log-ui`（前端）在线上环境的部署步骤。

## 1. 基础设施环境要求

建议在目标服务器上安装以下软件环境：
- **Node.js**: v18.x 或以上版本（用于运行前后端构建和后端服务）
- **MySQL**: v8.0.x 或以上版本（用于数据存储）
- **Nginx**: v1.20 或以上版本（用于前端静态资源托管及反向代理）
- **PM2**: 推荐用于 Node.js 后端进程管理的全局工具（可通过 `npm install -g pm2` 安装）

---

## 2. 数据库部署 (MySQL)

1. 登录目标 MySQL 服务器。
2. 创建项目所需数据库。
   ```sql
   CREATE DATABASE db_DevWorkLog CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 导入初始数据脚本。
   - 找到后端目录下的初始化脚本：`dev-work-log-server/init.sql`。
   - 将其在 `db_DevWorkLog` 数据库中执行以创建所需的表结构。
   - （视项目更新情况，若有增量更新，请酌情一并执行 `过程sql.txt` 中的 SQL 代码）。

---

## 3. 后端服务部署 (dev-work-log-server)

后端采用 Node.js + Express 架构。

### 3.1 传输代码
将本地的 `dev-work-log-server` 目录的所有文件（排除 `node_modules`）上传到服务器的指定目录（如 `/opt/dev-work-log/server`）。

### 3.2 安装依赖
在服务器端进入后端目录进行依赖安装：
```bash
cd /opt/dev-work-log/server
npm install --production
```

### 3.3 配置环境变量
编辑或新建 `.env` 文件，根据生产环境的信息配置相关数据库连接信息。
```env
PORT=3000
DB_HOST=127.0.0.1      # 替换为线上数据库地址
DB_PORT=3306
DB_USER=root           # 线上数据库账号
DB_PASSWORD=your_pwd   # 线上数据库密码
DB_NAME=db_DevWorkLog  # 数据库名
```

### 3.4 启动服务
建议使用 PM2 守护进程来运行保证稳定性：
```bash
pm2 start server.js --name "dev-work-log-server"
pm2 save
pm2 startup
```
执行后可以通过 `pm2 logs dev-work-log-server` 查看日志，确认连接数据库是否成功，以及控制台是否打印 `🚀 Server is running on port 3000`。

---

## 4. 前端页面部署 (dev-work-log-ui)

前端采用 Vue 3 + Vite 构建，生产环境仅需托管静态文件。

### 4.1 构建前端静态资源树
由于前端通过 `.env.production` 已经配置了 `VITE_API_URL=/api/v1`，此时打包会自动使用相对路径（利于交给 Nginx 拦截）。
你可以选择在**本地主机构建**，避免将代码逻辑留在生产服务器上：
```bash
cd dev-work-log-ui
npm install
npm run build
```
构建成功后，会在目录下生成一个 `dist` 文件夹，这就是最终需要在服务器上发布的前端交付物。

### 4.2 上传静态文件
将打包产生的 `dist` 文件夹上传至服务器目标路径，例如 `/usr/share/nginx/html/dev-work-log/`。

---

## 5. Nginx 反向代理配置 (关键)

配置 Nginx 来处理跨域、静态文件指向以及前端路由的 History 模式问题。

1. 在服务器 `/etc/nginx/conf.d/` 下新建一份配置文件，例如 `dev-work-log.conf`：
   ```nginx
   server {
       listen 80;
       server_name devworklog.yourdomain.com; # 修改为你的实际域名或服务器 IP

       # 前端静态文件路由
       location / {
           root /usr/share/nginx/html/dev-work-log; # 指向你上传的前端 dist 目录
           index index.html index.htm;
           try_files $uri $uri/ /index.html; # Vue Router History 模式的必需配置，防止 404
       }

       # 后端 API 接口反向代理
       location ^~ /api/ {
           # 注意此处的端口需要和 server 环境的 .env 中配置的 PORT 保持一致
           proxy_pass http://127.0.0.1:3000/api/;
           
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           
           # (可选) 允许长连接
           proxy_http_version 1.1;
           proxy_set_header Connection "";
       }
   }
   ```
2. 验证并重载 Nginx 配置：
   ```bash
   nginx -t
   nginx -s reload
   ```

## 6. 上线验证清单
- 访问首页 (`http://域名或IP`) ，检查页面 JS/CSS 静态资源是否正常加载，不报 404。
- 刷新页面，验证 Vue Router 服务端 History 跳转是否正常工作。
- 登录或请求列表数据，打开 F12 Network 控制台，检查发往 `/api/v1/...` 的接口请求是否得到响应。
- 检查后端 `pm2` 日志内是否有未捕获异常抛出。
